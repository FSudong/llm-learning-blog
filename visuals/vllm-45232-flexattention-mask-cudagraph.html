custom mask 不在 forward 现场决定，而是提前进入 FlexAttention metadata

GitHub PR #45232

先认识 custom mask

FlexAttention 的 logical_mask_mod 是一个函数，用 query/key 的逻辑位置判断能不能互相 attend。

def windowed_causal_mask_mod(b, h, q_idx, kv_idx):
    return (kv_idx <= q_idx) & (q_idx - kv_idx < 4)
kv <= q + 只看最近 4 个 = windowed causal mask

旧路径的问题：full graph 不喜欢 forward 里临时决定 mask

eager 路径

forward -> 查 layer mask -> 执行

可以每层临时查，代价是调度更动态。

full CUDA Graph

capture 一次 -> replay 多次 -> 不想 forward 动态变

mask 这类控制逻辑最好提前进 metadata。

新路径：初始化时收集，build 时写入 metadata

1FlexAttentionMetadataBuilder.__init__constructor
2get_layers_from_vllm_config(..., Attention, layer_names)layers
3_maybe_get_custom_mask_mod 检查所有层 mask 是否一致check
4build() 把 logical_mask_mod 和 sliding_window 写入 metadatametadata

为什么 alternating custom masks 先报错

允许

layer0 mask A layer1 mask A layer2 mask A

所有层同一个 custom mask，metadata 可以稳定表达。

拒绝

layer0 mask A layer1 mask B layer2 mask A

full graph 下任意 custom mask 交替暂不支持，PR 选择明确报错。

mask_mods = {
    getattr(layer, "logical_mask_mod", None)
    for layer in layers.values()
}
if len(mask_mods) > 1:
    raise ValueError(
        "cannot use alternating mask mods w/ full CUDA graphs"
    )

最后背这张卡

目标custom FlexAttention mask 也能使用 full CUDA Graph。
关键mask 和 sliding window 提前写进 metadata。
限制full graph 下不同层交替 custom mask 暂不支持。
