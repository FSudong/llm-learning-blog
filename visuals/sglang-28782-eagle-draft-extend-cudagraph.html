SGLang #28782：EAGLE draft-extend 进入 FlashInfer CUDA Graph

固定 qo layout，提前准备 wrapper，移除一处 D2H 同步

GitHub PR #28782

先看一句话版

EAGLE draft-extend 原来在 FlashInfer backend 下还有动态布局和同步点。PR 把每个请求的 query row 固定成 padded tree width，让这个阶段可以被 CUDA Graph capture/replay。

EAGLE draft -> draft-extend -> fixed qo_indptr -> FlashInfer graph replay

旧路径为什么不适合 graph

动态 query layout

num_accept_tokens -> cumsum -> qo_indptr 每轮变

每个请求真实接受几个 token 是运行时结果，query row 边界不稳定。

GPU 到 CPU 同步

prefix_lens on GPU -> sum().item() -> CPU 等 GPU

.item() 会把 GPU 上的小结果读回 CPU，decode 循环里很伤。

新路径：把动态变固定

旧 qo_indptr

qo_indptr[1:] = torch.cumsum(
    self.num_accept_tokens,
    dim=0,
)
req0: 2 行 req1: 5 行 req2: 1 行

新 qo_indptr

qo_indptr = torch.arange(
    0,
    (bs + 1) * num_tokens_per_req,
    step=num_tokens_per_req,
)
req0: 固定 4 行 req1: 固定 4 行 req2: 固定 4 行
固定不是让所有 padding token 都有效，而是让 tensor shape 和内存布局稳定。

FlashInfer backend 里新增的 graph 分支

1_prepare_cuda_graph_metadata 看到 is_draft_extend_v2()capture
2创建 causal paged prefill wrapper，不使用 custom maskwrapper
3按 batch size 存到 draft_extend_cuda_graph_metadata[bs]cache
4init_forward_metadata_out_graph replay 时复用 wrapperreplay

移除 D2H 同步的小改动

# old
paged_kernel_lens = prefix_lens
paged_kernel_lens_sum = (
    paged_kernel_lens.sum().item()
)
# new
if extend_prefix_lens_cpu is not None:
    paged_kernel_lens_sum = sum(extend_prefix_lens_cpu)
else:
    paged_kernel_lens_sum = paged_kernel_lens.sum().item()
CPU 已经知道的数字，就不要从 GPU 再读一遍。

限制条件

FlashInferAttnBackend + speculative_token_map is None = 允许 graph
FlashInferAttnBackend + FR-Spec token_map = fallback eager

PR 给出的效果

实验结果
Llama-2-7B + EAGLE topk=4，request-rate=8online median E2E latency 约 10% 改善
accept length基本不变，说明收益来自执行开销降低，不是投机接受率变化

最后背这张卡

目标FlashInfer draft-extend 支持 CUDA Graph。
核心改动用固定 num_tokens_per_req 生成 qo_indptr。
额外收益用 CPU 已知 prefix lens 替代 sum().item()。
